## 1. 포트원 회원가입 하기

포트원 관리자 콘솔은 **별도 계약 없이 무료로** 회원가입이 가능합니다.

  - 포트원 콘솔 화면에서 [가입하기]를 클릭해 주세요. [→ 포트원 콘솔 바로 가기](https://admin.portone.io)
  <Figure src={image1} width="480" caption="로그인 화면 예시"/>

  - 포트원에서 사용할 이메일 주소 및 비밀번호를 입력한 후 [가입]을 눌러주세요.
  <Figure src={image2} width="480" caption="회원가입 화면 예시"/>

  - 가입 완료 후 입력하신 이메일로 인증 메일이 발송됩니다. 메일 인증 전까지 로그인이 불가능하오니 꼭 인증 후 로그인을 시도해 주세요.
  <Figure src={image3} width="480" caption="인증 메일 내용 예시"/>

  <Hint style="info">
  메일을 받지 못하신 경우 스팸메일함을 꼭 확인해주세요.
  </Hint>

<br />
## 2. 전자결제 신청하기

비지니스 정보(사업자 정보) 등록 후 이용할 결제대행사 및 결제수단 신청합니다.

  - 로그인 후 [홈]화면에서 왼쪽 상단에 [전자결제 신청]을 클릭해 주세요.
  - 약관 동의 후 [시작하기!]를 클릭해 주세요.
  <Figure src={image4} width="480" caption="약관동의 화면 예시">

  - [비지니스 인증] 정보를 단계에 따라 입력합니다.
  - [전자결제 신청] 정보를 단계에 따라 선택합니다.

<Hint style="info">
  전자결제 신청 전 테스트를 하고싶으신 분은 아래 결제대행사 채널 설정하기를 참고하신 후 테스트 환경으로 채널 추가하여 테스트를 진행하실 수 있습니다.
</Hint>

## 3. 결제대행사 채널 설정하기

결제 연동 시 사용할 결제대행사 채널을 추가합니다.

  - 

  
포트원 회원가입 이후 관리자 페이지 내 **결제 연동** **->** **테스트 연동 관리** 탭에서 연동하고자 하는 PG사를 선택합니다.

**테스트 연동 관리** 메뉴에서는 포트원에서 지원하는 각 PG사별 테스트 설정을 진행할 수 있습니다.

<Figure
  src="/gitbook-assets/ko/image (74) (1).png"
  caption="결제연동 -> 테스트 연동 관리"
/>

<Hint style="info">
각 PG사별 **테스트 설정 및 실 계정 설정 방법**이 **상이**합니다. 자세한 PG사별 설정 방법은 [**링크**](2-pg/readme)를 통해 확인하세요.

</Hint>

## 3. 연동정보 확인하기

결제창 연동 및 API 호출 시 필요한 연동정보를 확인할 수 있습니다.

1. [**포트원 관리자콘솔**](https://admin.portone.io) 로그인 -> **상점 ・ 계정 관리** -> **내 식별코드 ・ API Keys** 선택
2. **가맹점 식별코드** / **REST API Key** / **REST API Secret** 확인

<Hint style="danger">
해당 정보는 **결제내역을 컨트롤**할 수 있는 **민감 정보**이기 때문에 외부로 유출되지 않도록

유의하시기 바랍니다.

</Hint>

<Tabs>
<Tab title="가맹점 식별코드">
* 포트원에서 관리자 콘솔 **계정당 고유하게 부여되는 값** 입니다.
* 결제창 호출 시 **객체 초기화**에 사용하는 고유 코드입니다.
* **기술 문의** 시 해당 코드값을 공유해 주시면 빠르게 문제 해결을 도와드릴 수 있습니다.

</Tab>

<Tab title="REST API 키">
* API 호출시 **Access Token** 발급을 위해 필요한 필수 값입니다.
* 해당 값은 **외부로 유출되지 않도록 각별히 유의**하셔야 합니다.

</Tab>

<Tab title="REST API secret">
* API 호출시 **Access Token** 발급을 위해 필요한 필수 값입니다.
* 해당 값은 재 발급이 가능하며 주기적으로 재 발급을 통해 보안을 높일 수 있습니다.
* 해당 값은 **외부로 유출되지 않도록 각별히 유의**하셔야 합니다.

</Tab>
</Tabs>
